# Install

The `@payrail-app/sdk` package wraps `fetch()` and handles the x402 payment
handshake on 402 responses. Your agent code stays a normal HTTP client —
Payrail signs the on-chain transaction on your behalf when a paid API
asks for payment.

## Requirements

- **Node 18 or later.** The SDK uses the native `fetch` API. On older
  runtimes, inject a polyfill via the `fetch` constructor option.
- **A Payrail account.** Sign in with Google or email at
  [payrail.sh](https://payrail.sh). The signup flow provisions an agent
  on your first top-up — see [Quickstart](/docs/agents/quickstart).

## Install the SDK

During the private beta the SDK is distributed via GitHub Releases. We'll
publish to npm at launch.

<CodeBlock lang="bash" filename="terminal">
{`npm install https://github.com/mayank-0789/payrail/releases/download/v0.1.0/payrail-sdk-0.1.0.tgz

# or pnpm:
pnpm add https://github.com/mayank-0789/payrail/releases/download/v0.1.0/payrail-sdk-0.1.0.tgz

# or yarn:
yarn add https://github.com/mayank-0789/payrail/releases/download/v0.1.0/payrail-sdk-0.1.0.tgz`}
</CodeBlock>

<Callout kind="tip" title="Post-launch">
  Once published to npm the one-liner becomes{" "}
  <code>npm install @payrail-app/sdk</code>. Everything else in these docs
  stays identical.
</Callout>

## Verify

A minimal sanity check — this should print the installed version:

<CodeBlock lang="bash" filename="terminal">
{`node -e "console.log(require('@payrail-app/sdk/package.json').version)"`}
</CodeBlock>

You're ready. The [Quickstart](/docs/agents/quickstart) gets you to a
first paid call in about three minutes.
