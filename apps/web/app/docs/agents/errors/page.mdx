# Error handling

The SDK throws `PayrailError` with a typed `code` for every failure mode.
Switch on `err.code` for user-facing responses, or rethrow unknown codes.

## The pattern

<CodeBlock lang="ts" filename="agent.ts">
{`import { Payrail, PayrailError } from "@payrail-app/sdk";

const agent = new Payrail({
  apiKey: process.env.PAYRAIL_KEY!,
  baseUrl: process.env.PAYRAIL_BASE_URL!,
});

try {
  const res = await agent.fetch(url);
  return await res.json();
} catch (err) {
  if (err instanceof PayrailError) {
    switch (err.code) {
      case "over_cap":
        // Monthly spend cap hit. Raise it in the dashboard or wait
        // for the calendar-month reset.
        throw new Error("This agent has no budget left this month.");
      case "agent_inactive":
        // Agent paused or cancelled in the dashboard.
        throw new Error("Agent is inactive.");
      case "invalid_token":
        // API key revoked, rotated, or mistyped.
        throw new Error("Payrail API key invalid — rotate in dashboard.");
      case "invalid_challenge":
        // The merchant sent a malformed 402 — not your fault.
        throw new Error("Merchant payment request was invalid.");
      case "signing_failed":
        // Umbra or Solana failed to sign. Safe to retry.
        throw new Error("Payment signing failed — retry in a moment.");
      case "network_error":
        // Couldn't reach Payrail backend.
        throw new Error("Payrail unreachable — retry in a moment.");
      default:
        throw err;
    }
  }
  throw err;
}`}
</CodeBlock>

## When to retry

The SDK doesn't retry on its own — that's your call, and depends on
whether the operation is idempotent. A safe pattern:

<CodeBlock lang="ts">
{`const RETRYABLE = new Set<string>([
  "signing_failed",
  "server_error",
  "network_error",
]);

async function callWithRetry(url: string, attempts = 2): Promise<Response> {
  try {
    return await agent.fetch(url);
  } catch (err) {
    if (
      err instanceof PayrailError &&
      RETRYABLE.has(err.code) &&
      attempts > 0
    ) {
      await new Promise((r) => setTimeout(r, 500));
      return callWithRetry(url, attempts - 1);
    }
    throw err;
  }
}`}
</CodeBlock>

`over_cap`, `agent_inactive`, and `invalid_token` are intentionally NOT
retryable — retrying them just wastes round-trips.
