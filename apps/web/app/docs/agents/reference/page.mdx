# API reference

Every export from `@obscura-app/sdk`, one section per symbol.

## `class Payrail`

The agent-side client. Wraps `fetch` and handles the x402 payment
handshake transparently.

<CodeBlock lang="ts">
{`import { Payrail } from "@obscura-app/sdk";

const agent = new Payrail(options: PayrailOptions);`}
</CodeBlock>

### `agent.fetch(url, init?)`

Behaves like `fetch`. On a 402 response with a `PAYMENT-REQUIRED` header,
signs via Payrail + retries the request with a `PAYMENT-SIGNATURE`
header. Returns the final `Response`.

Throws `PayrailError` on any failure — see [Error handling](/docs/agents/errors).

## `class PayrailError`

Typed error thrown by `agent.fetch()`.

<CodeBlock lang="ts">
{`import { PayrailError } from "@obscura-app/sdk";

try {
  await agent.fetch(url);
} catch (err) {
  if (err instanceof PayrailError) {
    console.log(err.code);     // PayrailErrorCode
    console.log(err.message);  // human string
  }
}`}
</CodeBlock>

### `PayrailError.code: PayrailErrorCode`

String literal union of every error the SDK can surface. See the
[error-code table](/docs/agents/errors#every-error-code) for each
code's meaning.

## Protocol-level notes

- **Signatures are bound to the challenge.** Each `PAYMENT-SIGNATURE` is
  generated from the merchant's specific `PAYMENT-REQUIRED` payload.
  Replaying a signature against a different challenge always fails
  verification.
- **Payrail is never in the request path for paid responses.** Only
  `/api/x402/sign` sits between you and the merchant, and only during
  the brief signing window. Your response data flows agent ↔ merchant
  directly.
- **SOL is never required on the agent side.** PayAI co-signs as
  fee-payer. Your agent wallet only ever needs USDC.
