# Install

The `@payrail/merchant-sdk` package gives you one Express-style
middleware that handles x402 payment challenges + settlement. Any route
it wraps becomes a paid endpoint: the middleware returns 402 until the
client pays, verifies + settles on Solana via the PayAI facilitator,
then hands off to your downstream handler.

## Requirements

- **Node 18 or later** and any framework with Express-style middleware —
  Express itself, Fastify (via `fastify-express`), Hono's adapter, or
  similar.
- **A Payrail merchant account.** [Sign up](/merchants) to get a managed
  Solana payout wallet — no crypto-native tooling required.

## Install the SDK

During the hackathon the SDK is distributed via GitHub Releases. We'll
publish to npm at launch.

<CodeBlock lang="bash" filename="terminal">
{`npm install https://github.com/payrail/payrail/releases/download/v0.1.0/payrail-merchant-sdk-0.1.0.tgz express

# or pnpm:
pnpm add https://github.com/payrail/payrail/releases/download/v0.1.0/payrail-merchant-sdk-0.1.0.tgz express

# or yarn:
yarn add https://github.com/payrail/payrail/releases/download/v0.1.0/payrail-merchant-sdk-0.1.0.tgz express`}
</CodeBlock>

<Callout kind="tip" title="Peer dependency">
  `@solana/web3.js` is a peer dep (expected at `>=1.98.4`). If your
  project doesn't already have it, install it alongside. We don't bundle
  it to keep your dependency tree deduplicated.
</Callout>

## Verify

<CodeBlock lang="bash" filename="terminal">
{`node -e "console.log(require('@payrail/merchant-sdk/package.json').version)"`}
</CodeBlock>

You're ready. The [Quickstart](/docs/merchants/quickstart) gets you a
settled payment in about five minutes.
